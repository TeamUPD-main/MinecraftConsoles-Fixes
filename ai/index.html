Minecraft Consoles Fixes Project Documentation

Welcome to the developer documentation for the Minecraft Consoles Fixes project. This project aims to maintain and improve the C++ codebase for various console versions of Minecraft.

This documentation serves as a comprehensive resource for understanding the project's structure, core components, and how to effectively contribute to its development.

Project Overview

The project is structured into several key components:

  • Minecraft.World: Contains the core game logic, including physics, entities, items, blocks, and data management.
  • Minecraft.Client: Handles all client-side operations, such as user interface, rendering, input processing, and platform-specific client integrations.
  • Minecraft.Server: Manages server-side logic, including multiplayer sessions, access control (banning and whitelisting), and persistent data storage.
  • Common: Provides shared utilities, definitions, and framework components used across Client, World, and Server layers.
  • 4JLibs, Durango, Orbis: These directories encapsulate platform-specific abstractions and implementations for Xbox (Durango) and PlayStation (Orbis) consoles.
