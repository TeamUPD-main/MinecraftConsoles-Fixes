20/* This sets how large the memory pool for a given resource types is, and how many handles
21 GDraw should allocate for it. GDraw keeps track of allocations in each pool, and will free

24 Returns 1 if value successfully changed, 0 on error.
25 You need to call IggyPlayerFlushAll on all active Iggys before you do this to make

27 You also need to call IggyFlushInstalledFonts if you have any installed fonts.

30IDOC extern GDrawFunctions * gdraw_D3D_CreateContext(IDirect3DDevice9 *dev, S32 w, S32 h);
31/* Creates a GDraw context for rendering using D3D. You need to pass in the D3D device
32 and the width/height of the content you're displaying.
33
34 The width/height is used solely for sizing internal rendertargets. They will be
35 allocated to the larger of this size and the size of any rendered tiles (with padding).
36 In other words, you can pass in (0,0) and the rendertargets will be allocated to the
37 right size. However, if you draw multiple Iggy files or tiles of different sizes,
38 they might first be allocated too small; it's best to pass in the correct size initially
39 to avoid unnecessary allocation/deallocation of too-small rendertargets.
40
41 There can only be one D3D GDraw context active at any one time.
42
43 If initialization fails for some reason (the main reason would be an out of memory condition),
44 NULL is returned. Otherwise, you can pass the return value to IggySetGDraw. */
45
