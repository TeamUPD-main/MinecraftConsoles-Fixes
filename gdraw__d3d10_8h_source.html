29IDOC extern GDrawFunctions * gdraw_D3D10_CreateContext(ID3D10Device *dev, S32 w, S32 h);
30/* Creates a GDraw context for rendering using D3D. You need to pass in the D3D device
31 and the width/height of render target textures.
32
33 The width/height is used solely for sizing internal rendertargets. They will be
34 allocated to the larger of this size and the size of any rendered tiles (with padding).
35 In other words, you can pass in (0,0) and the rendertargets will be allocated to the
36 right size. However, if you draw multiple Iggy files or tiles of different sizes,
37 they might first be allocated too small; it's best to pass in the correct size initially
38 to avoid unnecessary allocation/deallocation of too-small rendertargets.
39
40 There can only be one D3D GDraw context active at any one time.
41
42 If initialization fails for some reason (the main reason would be an out of memory condition),
43 NULL is returned. Otherwise, you can pass the return value to IggySetGDraw. */

96IDOC extern void RADLINK gdraw_D3D10_GetResourceUsageStats(gdraw_d3d10_resourcetype type, S32 *handles_used, S32 *bytes_used);
97/* D3D only: Get resource usage stats for last frame.
98 This can be used to get an estimate of how much graphics memory got used by GDraw
99 during the last frame.
100
101 For the dynbuffer, this always returns 0 in handles_used and the *size of the largest
102 single allocation* in bytes_used. It needs to be sized so that this allocation fits;
103 make it smaller and it won't work, but if you make it much larger (say more than 2x
104 as big), it's just a waste of memory. That said, we still recommend to make it no
105 smaller than 64k, and the default is 256k.
106
107 Caveat: This counts the number of bytes that GDraw knows about. 3D hardware usually
108 has its own management overhead, alignment requirements, allocation granularity
109 and so on. In short, this is not an accurate estimate of how much memory is actually
110 used by the GPU - it is a lower bound, though, and makes for a useful ballpark estimate. */
