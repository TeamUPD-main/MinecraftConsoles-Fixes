174/* Specifies a type of "special" blend mode, which is defined as one
175 that has to read from the framebuffer to compute its effect.
176
177 These modes are only used with a 1-to-1 textured quad containing
178 the exact output data in premultiplied alpha. They all need to
179 read from the framebuffer to compute their effect, so a GDraw
180 implementation will usually need a custom path to handle that.
181 Users will not warn in advance whether you're going to need this
182 operation, so implementations either need to always render to a
183 texture in case it happens, or copy the framebuffer to a texture
184 when it does.
185
186 Note that $(gdraw_blendspecial::GDRAW_BLENDSPECIAL_erase) and
187 $(gdraw_blendspecial::GDRAW_BLENDSPECIAL_alpha_special) are unique
188 among $gdraw_blendspecial modes in that they may not actually need
189 to be implemented with the destination input as a texture if
190 the destination buffer doesn't have an alpha channel. */
191
